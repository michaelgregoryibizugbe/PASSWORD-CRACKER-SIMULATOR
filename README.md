# PASSWORD-CRACKER-SIMULATOR Documentation

## Password Strength
Password strength refers to the effectiveness of a password in resisting various types of attacks. A strong password typically has the following characteristics:

- **Length**: Longer passwords are generally stronger. Aim for at least 12-16 characters.
- **Complexity**: Use a mix of upper and lower case letters, numbers, and symbols.
- **Unpredictability**: Avoid common words, phrases, and easily guessable information such as birthdays.

## Types of Attacks

### Brute Force Attack
A brute force attack involves systematically checking all possible combinations of characters until the correct password is found. It's the simplest form of password attack.

- **Advantages**: Guarantees finding the password eventually.
- **Disadvantages**: Time-consuming, especially for long and complex passwords.

### Dictionary Attack
A dictionary attack uses a prearranged list of potential passwords, typically derived from common words, phrases, and previously compromised passwords. This method is faster than brute forcing since it doesn't check every possible combination.

- **Advantages**: Quicker than brute force for weak passwords.
- **Disadvantages**: Ineffective against strong, unique passwords.
